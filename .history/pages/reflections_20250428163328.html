Reflection on Hackathon Project: LIDARcade

Event: HackPSU Date: March 2025 Category: Competition

Creating the DevPost article for our LIDARcade project demonstrated my ability to communicate our technical innovations to both judges and the tech community. This documentation required me to explain the technical implementation of AR technology, LiDAR sensors, and machine learning components while simultaneously presenting our project's social impact and market potential. I learned to balance technical accuracy with storytelling, presenting our hackathon project in a way that was accessible to technical and non-technical audiences alike. Writing this piece honed my skills in concise explanation, as the DevPost format was restricted with space limitations. For potential employers, this sample showcases my ability to emphasize technical propositions and convey engineering achievements into business and social benefits. The project's award-winning status validates the effectiveness of my technical communication approach.

Reflection on Web Development: Simple-Subscription-Tracker

Course: CMPSC 221 Date: November 2023 Category: Course Reflection

Developing the README.md file for my Simple Subscription Tracker project highlighted my understanding that good documentation is essential for software production and collaboration. This technical writing sample demonstrates my ability to create comprehensive yet concise documentation for a web application, covering everything from installation instructions to feature explanations and API documentation. Through this process, I learned to adjust to the needs of different readers—from developers who might want to contribute to the codebase to users seeking to understand the application's functionality. I structured the document with a hierarchical organization, used code examples to illustrate implementation details, and created visual aids to enhance comprehension. For potential employers, this sample demonstrates my commitment to documentation as a fundamental part of the development process, as opposed to an afterthought. It showcases my ability to make technical projects accessible and maintainable by others, a crucial skill for collaborative software development.

Reflection on Instruction Set: Connecting an External Monitor

Course: ENGL 202C Date: February 2025 Related to: Technical Instructions Sample

My instruction set for connecting an external monitor showcases my user-centered approach to technical documentation. This project taught me to expect diverse user needs and potential points of confusion when creating instructional content. Through this assignment, I developed skills in creating clear, c instructions that accommodate various hardware configurations and user expertise levels. What sets this document apart is its comprehensive organization—from equipment identification to troubleshooting—and its accessibility to users with different technical backgrounds. For employers, this sample demonstrates my ability to create documentation that reduces support needs and enhances user experience, directly impacting product adoption and customer satisfaction. It also shows my attention to detail and systematic thinking, as I worked to create instructions that would be error-proof across multiple scenarios. I hope visitors see that I approach technical communication with empathy for the end user, understanding that even simple processes can be challenging for some users and that well-crafted documentation serves as an important extension of the product experience.
